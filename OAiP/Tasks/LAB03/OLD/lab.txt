	       " Начальный сеанс работы с Turbo C ".
	    Раздел 1. Общее знакомство со средой Turbo C.
       Основа Turbo C (TC) - быстрый и эффективный  компилятор языка СИ
       с легко изучаемой и просто используемой интегрированной системой
       программыных средств. В ТС можно создавать и запускать программы
       без раздельного вызова редактора текста, компилятора, компонов-
       щика. Все перечисленные компоненты встроены в ТС, а доступ к ним
       реализуется посредством выбора из меню.
	    Предположим, требуется разработать программу с именем PROG.
       Все, что необходимо сделать для этого, можно представить в виде
       следующей простой схемы:
	  ----> PROG.C ----> PROG.OBJ ----> PROG.EXE
	    Сначала нужно загрузить среду Turbo C(необходимо запустить
       файл N:\COMMON\TC\TC.EXE), затем создать файл PROG.C (см. п.1),
       затем в редакторе набрать текст соответствующей программы и записать
       его в этот файл(это выполняется с помощью пункта меню File подменю
       Save).Подробнее сожержимое пункта меню File рассмотрим ниже в п.1.

	п.1: После того, как вы загрузили среду Turbo C, на экране появит-
	     ся окно основного меню Turbo C, состоящее из: главного меню,
	     окна редактора, окна сообщений, и строки оперативной подсказки.
	     Для активизации главного меню  (для выхода  в  главное  меню
	     редактора) необходимо нажать <F10> или нажатие:
		    <Alt+(первая буква пункта меню(лат.шрифт))>.
	     Об этом напоминает нижня строка редактора. Перемещение по
	     пунктам главного меню и подменю осуществляется клавишами
	     со стрелками, а активизация их с помощью <Enter>.
	     При активизации любого пункта меню можно получить оперативную
	     подсказку, характеризующую  этот пункт(с помощью кл.<F1>).
	     Для выхода из главного меню в окно редактора используется <Esc>.

	Охарактеризуем режимы(пункты) главного меню:
	____________________________________________
	      -File - работа с файлами: загрузка, сохранение, выбор, созда-
		      ние и др.;  работа с директориями (каталогами): изме-
		      нение, просмотр;  выход из Turbo C  и временный выход
		      в DOS;
	      -Edit    -переход в режим редактирования файлов;
	      -Run и   -запуск программ, работа по отладке программ(поша-
	   Break/watch	говое выполнение программ, точка останова и др.) и
			работа с окном Watch;
	      -Compile -трансляция программы в объектный и выполняемый
			файлы;
	      -Project -указание файлов, включаемых в программу, и управ-
			ление проектом ее создания;
	      -Options -выбор параметров компилятора: модель памяти, диаг-
			ностика, окружение и др.; запись директориев вы-
			ходных, библиотечных и включаемых файлов; сохране-
			ние параметров компилятора и загрузка их из файла
			конфигурации TCCONFIG.TC;
	      -Debug   -выявление ошибок и работа с сообщениями об ошибках.
     Рассмотрим подробнее пункт  File главного меню:
	   LOAD или <F3> -загрузить с диска в окно редактора и начать редак-
			  тировать уже существующий файл;
	   PICK или <Alt+F3>-выбрать файл из списка редактируемых файлов
			     и загрузить его в окно редактора;
	   NEW- перейти к созданию нового файла имя которого по умолчанию
		будет NONAME.C;
	   SAVE или <F2> -сохранить файл, находящийся в окне редактора, на
			  диске;
	   WRITE TO  - записать, находящийся в окне редактора файл под
		       новым именем или на место существующего старого
		       файла;
	   DIRECTORY - вывод на экран списка файлов;
	   CHANGE DIR- изменение текущего(активного) директория;
	   OS SHELL -временный выход в ОС( для возврата в среду Turbo C
		     нужно набрать команду exit);
	   QUIT или <Alt+X>-выход из Turbo C в ОС.

	    После того как текст программы был набран и сохранен в файл
       выполняется  компиляция  программы  и  образуется объектный файл
       (PROG.OBJ).

       На  последней стадии после подключения компановщика строится выпол-
       няемый файл (PROG.EXE).Компиляция и компоновка программы в интегри-
       рованной среде TC.EXE обычно включают следующие шаги:
	а)-устанавливаются параметры окружения, чтобы компилятор и компонов-
	   щик знали, где (на каких дисках) им искать файлы;
	б)-программа загружается в редактор;
	в)-создается файл выполняемой программы.


       ----------------сведения к пункту а)-----------------------------
       Кроме того, должны быть определены места расположения стандартных
       и библиотечных файлов Turbo C и программы загрузчика. Эти действия
       называют УСТАНОВКОЙ ПАРАМЕТРОВ РАБОЧЕГО ОКРУЖЕНИЯ. Они выполняются
       в следующей последовательности:
	    -активизируется режим Options и подрежим Directories;
	    -в подчиненном меню выбирается пункт Include directories и
	     указывается место расположения включаемый файлов *.h;
	    -точно также выбирается пункт Output directory и указывается
	     месторасположение файлов программ с типом OBJ,EXE,MAP;
	    -в строке Library directory вводятся расположение файлов с
	     типом LIB;
	    -в строке Turbo C directory вводится месторасположение фай-
	     лов TCCONFIG.TC,TCHELP.TCH);
	    -в подчиненном меню выбирается подрежим Store options;
	    -после появления на экране сообщения о файле конфигурации
	     TCCONFIG.TC нажимается клавиша <Enter>. В результате этот
	     файл будет сохранен там где вы указали и в нем сохранены
	     соответствующие параметры, используемые при последующей
	     загрузке TC.EXE.
      Что же будет, если эти параметры, указанные в п.1 не устанавливать?
      Тогда по умолчанию все файлы(*.h,OBJ,EXE,LIB) должны находиться на
      текущем диске(это либо диск с которого загружается программа TC.EXE
      или диск, который устанавливается активным в подменю Change dir в
      меню File).



      --------------------сведения к пункту б)---------------------------

	  Следующий шаг - загрузка в окно редактора программы(см.описание
      пункта File главного меню).




     ---------------------сведения к пункту в)--------------------------
	   Следующий шаг предполагает создание выполняемого файла. Для
      этого необходимо оттранслировать исходный файл PROG.C(подменю
      Compile to OBJ пункта меню COMPILE) и скомпоновать получившийся
      объектный модуль PROG.OBJ со стандартными библиотеками и файлом
      загрузчика.
      Охарактеризуем пункт меню COMPILE более подробно:
      Compile to OBJ - команда трансляции исходного текста в объектный
		       файл. В строке этого подпункта находится специ-
		       фикация файла,  получаемого в результате тран-
		       сляции(например o:\PROG.OBJ).
      Make EXE file -  команда построения выполняемого файла.
      Link EXE file -  команда компоновки текущего объектного файла с
		       библиотеками *.LIB, в результате появляется
		       новый выполняемый файл;
      Build all     -  команда перетрансляции всех файлов проекта,
		       независимо от даты их образования;
      Primary C file-  команда для указания имени файла, который должен
		       транслироваться в объектный код при выполнении
		       команды.
      Запуск программы на выполнение осуществляется с помощью команды Run
      из главного меню.


      ПРИМЕЧАНИЕ:
      ___________
      ___________
      После набора текста программы в окне редактора ее трансляция, компанов-
      ка, загрузка и выполнение могут быть реализованы совсем просто.
      Для этого достаточно сразу же выполнить команду Run из главного меню,
      и все требуемые режимы для создания файлов OBJ, EXE, запуск файла
      типа EXE на выполнение будут осуществляться автоматически.

	   Одним из сложных аспектов программирования всегда было нахожде-
      ние и исправление ошибок. Легкость их обработки является одной из
      причин побуждающих к использованию Turbo C.  Если в результате ком-
      пилящии были обнаружены ошибки на экране появится строка Press any key
      (нажмите любую клавишу). После выполнения рекомендуемых действий
      появится окно сообщений. При этом будут указаны как сообщения об ошиб-
      ке, так и вызвавшая его неправильная строка текств программы (в окне
      редактора). Если вы  хотите просмотреть другие ошибки воспользуйтесь
      клавишами со стрелками. Обратите внимание, что одновременно с пере-
      бором ошибок курсор в окне редактора постоянно перемещается на соот-
      ветствующую строку. Для просмотра ошибок можно использовать и клави-
      ши <F7> и <F8>(предыдущая и последующая ошибки).
	   Одним из преимуществ Turbo C является то, что он обеспечиваетъ
      раздельную компиляцию нескольких исходных файлов. Это достигается
      средствами создания проектов. При построении программы из нескольких
      исходных файлов необходимо указать компилятору, какие программы ис-
      пользуются.Предположим, что имеется два файла, первый из которых
      содержит основную программу (PROG1.C), а второй - подпрограмму
      (PROG2.C).Тогда можно создать проект, который будет содержать всего
      две строки с именами файлов:
		       PROG1
		       PROG2
      Назовем файл проекта PROG.PRJ.В отношении строк PROG1 и PROG2 следует
      напомнить, что для любого файла без расширения подразумевается тип С,
      кроме того последовательность указания файлов не имеет значения,
      влияя лишь на очередность их трансляции.
      Обратим внимание , что имя выполняемой программы будет устанавливаться
      по имени файла проекта. В проекте можно указывать и спецификацию файла,
      поэтому неважно где на диске располагаются файлы, входящие в проект.
      Например имя проекта PROG.PRJ. Имя проекта необходимо ввести в меню
      работы с проектом. Для этого надо выбрать подпункт Project name в
      пункте меню PROJECT и задать спецификацию соответствующего файла.
      Далее для создания выполняемого файла достаточно нажать <F9>.
      Запуск файла PROG.EXE на выполнение осуществляется командой Run
      главного меню.
      Охарактеризуем остальные подпункты пункта меню PROJECT:
      Break make on - команда, дающая возможность указать, после чего
		      должна прерваться работа: после трансляции файла,
		      в котром обнаружены ошибки,предупреждения,фаталь-
		      ные ошибки, или перед компановкой файла;
      Clear project - команда удаления имени из пункта Project name.
      Надо отметить, что окно редактора Turbo C можно использовать как
      обычный текстовый редактор. Рассмотрим ниже некоторые команды
      работы с блоками текста:
         -отметить начало\конец блока: Ctrl-K B\Ctrl-K K
         -скопировать блок           : Ctrl-K C
         -переместить блок           : Ctrl-K V
         -удалить блок               : Ctrl-K Y
         -выделить слово             : Ctrl-K T
         -скрыть\показать блок       : Сtrl-K H
         -записать\прочитать блок
          в\из файла                 : Ctrl-K W\Ctrl-K R
       Команды перемещения курсора (быстрого) :
         -в начало\конец строки      : Ctrl-Q S(Home)\Ctrl-Q D(End)
         -в верх\низ экрана          : Ctrl-Q E\Ctrl-Q X
         -в верх\низ файла           : Ctrl-Q R\Ctrl-Q C
         -последняя позиция курсора  : Ctrl-Q P
       Команды замены и вставки:
         -удалить строку             : Ctrl-Y
         -вставить строку            : Ctrl-N
       Прочие команды
         -поиск                      : Ctrl-Q F
         -поиск с заменой            : Ctrl-Q A
         -поиск с отметки места      : Ctrl-Q N
         -восстановить строку        : Ctrl-Q L

    С остальными командами текстового редактора можно ознакомиться
    изучив Help по среде Turbo C (который вызывается с помощью кла-
    виши F1).



	      Раздел 2.  В А Ш А   П Е Р В А Я   П Р О Г Р А М М А

	 После знакомства со средой Turbo C - начнем изучения языка Си.
	 Опередим события и сразу же напишем программу на языке Си:
	 программу, выводящую на экран слова: ПРИВЕТ ИЗ БРЕСТА.
	 Текст данной программы:
		      #include <stdio.h>
		       main()
			{
			   printf("ПРИВЕТ ИЗ БРЕСТА");
			 }

	  Пояснения: в первой строке программы указана команда препро-
	  цессора #include, включающая файл stdio.h, который описывает
	  библиотечную функцию printf.Однако, кроме функции printf в
	  нем описаны многие другие библиотечные функции ввода\вывода,
	  и чаще проще включить один файл, чем  помнить  о  множестве
	  строк с описаиями.
	       В языке Си любая программа состоит из одной или более
	  "функций" , задающих действия, которые нужно выполнить.
	  Различным функциям можно давать какие угодно имена. Среди
	  них есть одно особое - main; выполнение любой программы
	  начинается с функции main. В нашей программе в ней нет аргу-
	  ментов, поэтому их список выглядит так ().
	       Функция printf является библиотечной: она позволяет
	  вывести на экран дисплея текст ПРИВЕТ ИЗ БРЕСТА.Более под-
	  робно эту и другие функции языка Си будем изучать  на пос-
	  ледующих занятиях.
	       Договоримся разместить исходный текст программы на
	  диске O:\ и назвать его PRIV.C.Тогда для ее выполнения необ-
	  ходимо реализовать следующие действия:
	     -загрузить среду Turbo C,пользуясь сведениями полученными
	      из раздела 1 этой методички;
	     -набрать в редакторе текст вашей программы(согласно зада-
	      ния и варианта);
	     -сохранить текст на диске O:\;
	     -теперь когда программа набрана и сохранена, необходимо ее
	      оттранслировать и скомпоновать,запустить на выполнение.
	      О том, как это сделать, смотри выше (в разделе 1, подробно
	      рассмотри описание пункта меню Run);
